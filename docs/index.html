Problem

  • Students struggle to find timely peer help for doubts.
  • Study groups are hard to organize and maintain.
  • Senior guidance is scattered and difficult to request.
  • Project teammates, roles, and task ownership are unclear.
  • Real contributions are difficult to prove during reviews.

Solution

  • Anonymous doubts and peer help workflows.
  • Learning circles with resources and discussion.
  • Senior Connect for public-safe mentor guidance.
  • Project Mate Finder with secure team workspaces.
  • Task verification and contribution history in profiles.

Deep Dive

Project Mate Finder & Workspace

The strongest module of the MVP: students can form teams, control project access, share resources, assign work, submit deliverables, request extensions, and build verified contribution history.

Team Formation

Create project posts, add role slots, accept/reject applicants, and keep live capacity counts accurate.

Secure Workspace

Only project owners and active members can access private coordination, discussion, resources, and tasks.

Resource Library

Supports links, PDFs, images, documents, folders, code repos, and video previews with signed URLs.

Project Tasks

Team leads assign tasks with deadlines, attachments, priorities, and role tags.

Submission Review

Members submit work; owners verify or request changes. Verified work updates profile history.

Deadline Extensions

Members can request more time; leads approve/reject while preserving task history.

Security & Privacy

  • Supabase Row Level Security protects private tables.
  • Project workspace access is gated to owner/active members.
  • Past members and non-members cannot access private files.
  • Raw storage paths are hidden; signed URLs are used for files.
  • Private contact fields are gated by visibility settings.
  • Frontend uses anon key only; service role key is never exposed.

Run Locally

Developer setup

Use the README and Supabase setup guide for the complete database and storage configuration.

git clone https://github.com/maheshofficial0011/skillsaathi-campus-peer-learning.git
cd skillsaathi-campus-peer-learning
npm install
cp .env.example .env.local
npm run dev

# Verification
npx tsc -p tsconfig.app.json --noEmit
npm run build

Deployment

  • Live app runs on Vercel using the Vite preset.
  • Build command: npm run build.
  • Output directory: dist.
  • Required env vars: VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY.
  • Supabase Auth redirect URLs must include the Vercel URL and local dev URL.
